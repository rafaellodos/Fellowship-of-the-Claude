# What The Fellowship Is

---

Most people using AI assistants are doing it wrong. Not because they're using the wrong tools — because they're using the right tools without structure.

A single AI conversation has no memory of last week's decision. No awareness of the broader project. No way to bring a specialist to a problem without losing the context that made the problem legible. Every session starts from scratch. Every output is one-dimensional — the same voice, the same angle, whether you needed a strategist or a writer or someone to punch holes in your plan.

The Fellowship is a different approach.

---

## The idea

You have a team. Fifteen specialists, each with a distinct mandate, a governing principle, and a documented fatal flaw. A strategist. A builder. A writer. A researcher. A financial analyst. A security auditor. An ethics reviewer. A client relationship manager. An intelligence synthesiser. A few others.

They don't all show up at once. Gandalf — the orchestrator — reads the task, determines which specialists the job requires, deploys them, and synthesises their work into something you can actually use. You talk to Gandalf. The fellowship operates in the background.

The whole system lives on your local filesystem. No cloud. No database. No app required. Claude Code is the runtime — the fellowship gives it structure, memory, and a team.

---

## What it feels like

You open Claude Code in your fellowship folder and tell Gandalf what you're working on.

If it's a commercial decision, Harvey assesses the competitive frame before you make a move. If it's a piece of writing, Chinaski takes it and returns it in the right register. If you're about to do something consequential, Morty reviews the plan before it goes out, and Socrates interrogates the assumptions you didn't know you were making.

If you need to build something, Rick scopes it and builds it. If you need to understand the market, Archer researches and Killua turns the signals into a model you can actually decide from. If the economics need checking, The Wolf tells you whether the math survives contact with reality.

The session ends. Aragorn — your active working memory — records what happened. Next session, Gandalf knows where things stand.

Over time, the fellowship accumulates context. The decisions you made, the clients you talked to, the knowledge you built up — all of it lives in a three-layer memory system that agents draw from without you having to re-explain it every time.

---

## What it requires

A Claude Code subscription (Claude Pro, Max, or API). A terminal. That's it.

No server. No database. No setup beyond cloning a repo and answering five questions.

The system bootstraps itself. First time you open Claude Code in the folder, Gandalf runs a ten-minute intake — who you are, what your domains are, how you work — and builds your workspace from the answers. You never do setup again.

---

## Who this is for

People who work across multiple domains and need genuine specialisation, not the same general assistant responding to everything the same way.

Builders who want a technical partner, a strategic sounding board, and a financial reality-check without switching tools.

Writers who want the thinking done before the writing starts, and the writing reviewed by something that understands voice.

Consultants and founders who run several things at once and need the system to remember what each thing is and where it stands.

Anyone who has felt the gap between "AI assistant that can help with anything" and "AI partner that understands what I'm building."

---

## What it isn't

It isn't an app. There's no dashboard, no interface to log into, no service to subscribe to beyond Claude Code.

It isn't magic. The agents are as good as the context you give them and the clarity with which you direct them. Garbage in, garbage out — but the system is designed to reduce garbage in by asking the right questions before acting.

It isn't finished. This is a living system. The agents evolve as you use them. The memory deepens as you work. It gets more useful the more you put into it.

---

## The short version

Fifteen specialist agents. One orchestrator. Your local filesystem as the brain. A ten-minute setup that never needs to be repeated. A system that gets better the longer you use it.

That's the fellowship.
